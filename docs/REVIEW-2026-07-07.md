# Project review — 2026-07-07

A full-repo health review (code, tests, docs, tooling), with concrete
instructions. Items are ordered by leverage: fixing the top ones makes every
future agent session on this repo faster and safer. Verified against the
codebase on this date — typecheck was green and all 15 pure-bun tests passed
during the review.

## 1. Finish the Praxis rebrand (naming drift is everywhere)

`package.json` says `praxis`/`Praxis`/`com.alikimovich.praxis`, but the repo
dir, the GitHub remote (`alikimovich/dsgn`), README.md, CLAUDE.md, and all of
`docs/` still say "dsgn". Meanwhile lkmv.ch integration files are literally
named `dsgn-*` (`data-dsgn-source`, `.dsgn/`, `dsgn-source-plugin.cjs`).

**Instruction:** decide the blast radius of the rename and write it down in
`docs/CONTEXT.md`. Two sane options: (a) product name is Praxis, protocol/attr
names stay `dsgn` (cheapest — just fix the prose in README/docs); (b) full
rename including repo + attribute names (touches lkmv.ch and any other target
repo). Don't leave it half-done — agents keep tripping on which name to use.
CLAUDE.md has been updated (same date as this review) to state option (a) as
the interim reality.

## 2. Replace the package.json test mega-chains with a runner

`test` and `verify` are ~50-command `&&` chains, duplicated almost verbatim
between the two scripts. Costs: every new test must be appended to two giant
strings (easy to miss one — they've already drifted: `verify` = `test` + 3
live e2e tests); the first failure aborts the whole chain with no summary of
what else would have failed; `electron-vite build` runs redundantly; no way to
run "everything except sim tests" without hand-editing.

**Instruction:** add `test/run.mjs` with a manifest:

```js
const tiers = {
  unit:     ['pr-body', 'publish-message', 'project-key', /* … bun-runnable */],
  electron: ['smoke', 'menu-recents', 'open-preview', /* … need built app */],
  live:     ['agent-e2e', 'codex-e2e', 'sim-e2e'],   // need creds; self-SKIP
}
```

Run unit via `bun`, build once, run electron tests sequentially, collect
pass/fail/skip, print a summary table, exit non-zero if any failed (keep-going
by default, `--bail` optional). Then `"test": "node test/run.mjs unit electron"`,
`"verify": "node test/run.mjs all"`. Keep the individual `test:*` aliases only
for the ones actually used day-to-day.

## 3. Extract a shared test harness

All 55 `.mjs` tests re-derive the repo root and re-implement Playwright/
Electron launch, artifact paths, and DOM helpers (~6,200 lines total, a large
fraction boilerplate). **Instruction:** create `test/lib/harness.mjs` exporting
`launchApp()`, `openFixture(name)`, `shot(page, name)` (writes to
`test/artifacts/`), and the common wait/assert helpers. Migrate tests
opportunistically — whenever a test is touched, port it; don't do a big-bang
rewrite.

## 4. Add CI (there is none — no `.github/` at all)

The entire suite is agent-run-only today. The pure-bun tier + typecheck are
cheap and display-free. **Instruction:** add
`.github/workflows/ci.yml` running on push/PR: `bun install`,
`bun run typecheck`, unit tier from item 2. The Electron/Playwright tier can
run on a macOS runner later (the self-hosted `panda-mac` runner used by kitOS
is an option) — don't block on it; typecheck+unit in CI already catches the
regressions agents most often introduce.

## 5. Adopt one lint/format tool

There is no ESLint, no Prettier, no format script — style is whatever the last
agent emitted. **Instruction:** pick Biome (single fast tool, good TS/React
defaults) → `bun add -d @biomejs/biome`, `bunx biome init`, add
`"lint": "biome check src test"` and wire into CI. Run one repo-wide
`biome check --write` commit so future diffs are clean.

## 6. Split the god files (priority order)

- `src/renderer/src/App.tsx` (1,646) — extract the pane-layout/geometry logic,
  keyboard handling, and per-panel wiring into hooks/components; App should be
  composition only.
- `src/renderer/src/styles.css` (1,836) — post-Tailwind-migration this is
  legacy CSS coexisting with utilities. Move rules into the components that
  own them (or CSS modules / `@layer`), keep only tokens + true globals here.
- `src/main/props.ts` (1,189) and `props-svelte.ts` (701) — they mirror each
  other's splice/apply contract by design; extract the shared contract/types
  so the mirror stays honest.
- `src/main/simulator.ts` (1,169) — split device discovery / boot lifecycle /
  MJPEG bridge.
- `src/renderer/src/store.ts` (981) — consider zustand slices per domain
  (chat, preview, props, sessions).

Rule of thumb going forward: new module at ~500 lines, no exceptions for
"it's all related".
